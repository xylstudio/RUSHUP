# 🎯 คู่มือใช้งานฟีเจอร์ Estimated Duration (พร้อมเลือกหน่วยเวลา)

## สำหรับ Admin

### วิธีสร้างบริการใหม่พร้อม Estimated Duration

1. **เข้าสู่หน้าสร้างบริการ**
   - ไปที่ Dashboard > Admin > Services > Create Service

2. **กรอกข้อมูลพื้นฐาน (ขั้นตอนที่ 1)**
   - ชื่อบริการ
   - คำอธิบาย  
   - ราคาเริ่มต้น
   - **ส่วนระยะเวลาดำเนินงาน:**
     - Toggle "แสดงระยะเวลาดำเนินงาน" เปิด/ปิดตามต้องการ
     - ถ้าเปิด: กรอกจำนวน + เลือกหน่วยเวลา (ชั่วโมง/วัน)

3. **เลือกหมวดหมู่ (ขั้นตอนที่ 2)**
   - ระบบจะ auto-set การตั้งค่า estimated duration ตามหมวดหมู่:
     - **จัดสวน**: เปิด 7 วัน
     - **ออกแบบ**: เปิด 14 วัน
     - **ก่อสร้าง**: เปิด 30 วัน
     - **บำบัดต้นไม้**: เปิด 5 วัน
     - **บำรุงรักษา**: ปิด (แต่มี default 4 ชั่วโมง)
     - **ทำความสะอาด**: ปิด (แต่มี default 3 ชั่วโมง)
   - Admin สามารถเปลี่ยนแปลงได้ตามต้องการ

4. **ยืนยันและบันทึก**
   - ตรวจสอบ summary ในขั้นตอนสุดท้าย
   - ระยะเวลาและหน่วยจะแสดงเฉพาะเมื่อเปิดใช้งาน

### ตัวอย่างการใช้งาน

#### บริการออกแบบสวน
- ✅ **เปิด** Estimated Duration  
- 📅 **14 วัน** (ออกแบบ + revise + finalize)
- 💡 **เหตุผล**: ลูกค้าต้องการทราบระยะเวลาในการได้รับแบบ

#### บริการทำความสะอาดด่วน  
- ✅ **เปิด** Estimated Duration
- ⏰ **3 ชั่วโมง** (งานเร็ว ภายในวันเดียว)
- 💡 **เหตุผล**: ลูกค้าต้องการทราบว่าจะเสร็จภายในวันนี้

#### บริการดูแลสวนประจำเดือน  
- ❌ **ปิด** Estimated Duration
- 💡 **เหตุผล**: เป็นบริการต่อเนื่อง ไม่มีจุดสิ้นสุดที่ชัดเจน

#### บริการก่อสร้างสวน
- ✅ **เปิด** Estimated Duration
- 📅 **30 วัน** (ก่อสร้าง + ติดตั้ง + ทดสอบ)
- 💡 **เหตุผล**: ลูกค้าต้องการวางแผนการใช้งานพื้นที่

### การดูข้อมูลระยะเวลาดำเนินงาน

#### 1. ในรายการบริการ
- แสดงไอคอน 🕐 พร้อมระยะเวลาและหน่วย ใต้คำอธิบายบริการ
- ตัวอย่าง: "ระยะเวลา 3 ชั่วโมง" หรือ "ระยะเวลา 14 วัน"
- ปรากฏเฉพาะบริการที่ admin เปิดใช้งาน

#### 2. ในหน้ารายละเอียดบริการ  
- แสดงในส่วน "บริการที่รวมอยู่ในทุกแพ็กเกจ"
- มีไอคอน ClockIcon และข้อความชัดเจน
- รวมคำอธิบายเสริม "*ระยะเวลาโดยประมาณ อาจแตกต่างไปตามสภาพพื้นที่จริง"

#### 3. ในหน้าสรุปการสั่งซื้อ
- แสดงในส่วน "สรุปรายการ" 
- ช่วยลูกค้าตรวจสอบก่อนชำระเงิน

## คู่มือเลือกหน่วยเวลา

### การใช้ "ชั่วโมง" เหมาะกับ:
- **บริการรวดเร็ว** ที่เสร็จภายในวันเดียว
- **งานซ่อมแซมเล็กๆ** เช่น ตัดหญ้า, รดน้ำ
- **การทำความสะอาด** ที่ไม่ซับซ้อน
- **การตรวจสอบ** หรือ การประเมิน

**ตัวอย่าง:**
- ตัดหญ้า: 2 ชั่วโมง
- ทำความสะอาดใบไม้: 1.5 ชั่วโมง  
- ตรวจสอบระบบน้ำ: 1 ชั่วโมง

### การใช้ "วัน" เหมาะกับ:
- **โปรเจคใหญ่** ที่ใช้เวลาหลายวัน
- **งานออกแบบ** ที่ต้องใช้เวลาคิด
- **การก่อสร้าง** หรือปรับปรุงภูมิทัศน์
- **งานที่ต้องรอแห้ง** หรือ รอผลลัพธ์

**ตัวอย่าง:**
- ออกแบบสวน: 14 วัน
- สร้างสวนหินใหม่: 7 วัน
- ปลูกต้นไม้ใหญ่: 3 วัน

## Tips และ Best Practices

### สำหรับ Admin
- **เลือกหน่วยให้เหมาะสม** - งานเร็วใช้ชั่วโมง งานใหญ่ใช้วัน
- **ระยะเวลาให้สมเหตุสมผล** - พิจารณาจากงานจริงและเผื่อเวลาไว้
- **บริการซับซ้อนให้ระยะเวลานาน** - เช่น ก่อสร้าง 30 วัน, ออกแบบ 14 วัน
- **บริการต่อเนื่องไม่ต้องแสดง** - เช่น บำรุงรักษารายเดือน
- **อัปเดตตามฤดูกาล** - บางช่วงอาจต้องใช้เวลานานกว่าปกติ

### สำหรับ UX Design
- **ข้อมูลจำเป็นเท่านั้น** - ไม่ใส่ระยะเวลาทุกบริการ
- **ชัดเจนและเข้าใจง่าย** - แสดงหน่วยเวลาให้ชัดเจน
- **สอดคล้องกันทั้งระบบ** - ใช้ icon และสีเดียวกัน

## Database Schema Update

```sql
-- เพิ่มฟิลด์ has_estimated_duration และ estimated_duration_unit
ALTER TABLE services 
ADD COLUMN has_estimated_duration BOOLEAN DEFAULT FALSE,
ADD COLUMN estimated_duration_unit VARCHAR(10) DEFAULT 'days';

-- เพิ่ม constraint
ALTER TABLE services 
ADD CONSTRAINT check_estimated_duration_unit 
CHECK (estimated_duration_unit IN ('hours', 'days'));
```

## Troubleshooting

### ปัญหาที่อาจพบ

#### 1. Estimated Duration ไม่แสดงในหน้า Customer
- ✅ ตรวจสอบว่า admin เปิด toggle ใช้งานแล้วหรือไม่
- ✅ ตรวจสอบว่ากรอกจำนวนและเลือกหน่วยแล้วหรือไม่  
- ✅ ตรวจสอบฐานข้อมูลว่า `has_estimated_duration = true`

#### 2. หน่วยเวลาแสดงผิด
- ✅ ตรวจสอบ `estimated_duration_unit` ในฐานข้อมูล
- ✅ ตรวจสอบ `formatDuration` function ใน customer page

#### 3. Auto-setting ไม่ทำงาน
- ✅ ตรวจสอบ `SERVICE_CATEGORIES` มี defaultEstimatedDurationUnit
- ✅ ตรวจสอบ `handleInputChange` function
