# ระบบการแจ้งเตือนและการเปลี่ยนสถานะ - คู่มือทดสอบ

## 📋 รายการตรวจสอบ (Checklist)

### ✅ 1. ฝั่งพนักงาน (Staff Dashboard)
- [ ] เข้า `/dashboard/staff/measurements`
- [ ] เห็นรายการคำขอการวัดพื้นที่ที่มีสถานะ "รอดำเนินการ" (pending)
- [ ] มีปุ่ม "รับงาน" สำหรับคำขอที่ยังไม่มีผู้รับ
- [ ] กดปุ่ม "รับงาน" แล้วเห็นข้อความ "รับงานวัดพื้นที่เรียบร้อยแล้ว!"
- [ ] สถานะเปลี่ยนเป็น "มอบหมายงานแล้ว" (assigned)
- [ ] ใน console เห็นข้อความ "📢 Notification sent to customer"

### ✅ 2. ฝั่งลูกค้า (Customer Dashboard)  
- [ ] เข้า `/dashboard/customer`
- [ ] เห็นจำนวนการแจ้งเตือนใหม่ในการ์ด "การแจ้งเตือนใหม่"
- [ ] เห็นส่วน "การแจ้งเตือนล่าสุด" แสดงข้อความจากพนักงาน
- [ ] เห็นส่วน "สถานะการวัดพื้นที่" แสดงสถานะที่อัพเดทแล้ว
- [ ] ได้รับ browser notification (หากอนุญาต)

### ✅ 3. TopNavBar Notification
- [ ] เห็นปุ่มระฆัง (🔔) ใน TopNavBar
- [ ] มี badge แสดงจำนวนการแจ้งเตือนที่ยังไม่อ่าน
- [ ] คลิกที่ปุ่มระฆังแล้วเห็น dropdown แสดงรายการการแจ้งเตือน
- [ ] สามารถกด "อ่านทั้งหมด" ได้
- [ ] การแจ้งเตือนใหม่แสดง badge สีแดง

## 🔧 วิธีทดสอบ

### ขั้นตอนที่ 1: เตรียมข้อมูล
1. ให้ลูกค้าสร้างบ้านใหม่ที่ `/dashboard/customer/houses/add`
2. เลือก "ต้องการให้ทีมงานมาวัดพื้นที่ (ฟรี)"
3. กดบันทึก - จะสร้าง measurement request ใหม่

### ขั้นตอนที่ 2: ทดสอบการรับงาน
1. พนักงานเข้า `/dashboard/staff/measurements`
2. หาคำขอที่เพิ่งสร้าง (สถานะ pending)
3. กดปุ่ม "รับงาน"
4. ตรวจสอบ:
   - ข้อความ success แสดงขึ้น
   - สถานะเปลี่ยนเป็น assigned
   - Console แสดง "📢 Notification sent to customer"

### ขั้นตอนที่ 3: ทดสอบการแจ้งเตือน
1. ลูกค้าเข้า `/dashboard/customer`
2. ตรวจสอบ:
   - การ์ด "การแจ้งเตือนใหม่" แสดงจำนวน 1
   - ส่วน "การแจ้งเตือนล่าสุด" แสดงข้อความ
   - ส่วน "สถานะการวัดพื้นที่" แสดงสถานะ "มอบหมายงานแล้ว"
3. ตรวจสอบ TopNavBar:
   - ปุ่มระฆังมี badge สีแดง
   - คลิกแล้วเห็นการแจ้งเตือน

## 🐛 การแก้ไขปัญหา

### ปัญหาที่อาจพบ:
1. **ไม่เห็นการแจ้งเตือน**: ตรวจสอบ browser console ว่ามี error หรือไม่
2. **สถานะไม่เปลี่ยน**: ตรวจสอบ API call ใน Network tab
3. **RLS Error**: รัน `emergency-rls-fix.sql` ใน Supabase
4. **ไม่แสดงชื่อลูกค้า**: รัน `simple-fix-display-names.sql`

### การตรวจสอบ Console:
- Staff Dashboard ควรเห็น: "📢 Notification sent to customer"
- Customer Dashboard ควรเห็น: "✅ Retrieved measurement requests"
- TopNavBar ควรเห็น: event listeners สำหรับ 'newNotification'

## 📱 ข้อความแจ้งเตือนที่คาดหวัง:
```
"ทีมงานได้รับงานการวัดพื้นที่ที่ [ชื่อบ้าน] แล้ว จะติดต่อกลับเร็วๆ นี้"
```

- ระบบทำงานแบบ real-time ผ่าน CustomEvent
