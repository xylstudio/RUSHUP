# นโยบายความเป็นส่วนตัวและการคุ้มครองข้อมูลส่วนบุคคล (PDPA)

เวอร์ชัน: 1.0  
ปรับปรุงล่าสุด: 5 พฤษภาคม 2026

## 1. บทนำ

บริษัท Xylem Landscape และระบบดิจิทัลที่เกี่ยวข้อง รวมถึงเว็บไซต์ พอร์ทัลลูกค้า พอร์ทัลพนักงาน ระบบผู้ดูแล ระบบเวิร์กช็อป ระบบการแจ้งเตือน และการเชื่อมต่อกับ LINE, อีเมล, ระบบชำระเงิน และแผนที่ออนไลน์ ให้ความสำคัญกับการคุ้มครองข้อมูลส่วนบุคคลตามพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 และหลักการกำกับดูแลข้อมูลที่เหมาะสมกับการให้บริการจริง

นโยบายฉบับนี้อธิบายว่าเราเก็บ ใช้ เปิดเผย จัดเก็บ ระยะเวลาการเก็บรักษา และสิทธิของเจ้าของข้อมูลอย่างไร โดยอ้างอิงจากการทำงานจริงของระบบ Xylem Landscape

## 2. ขอบเขตของนโยบาย

นโยบายนี้ใช้กับข้อมูลส่วนบุคคลที่ได้รับผ่านช่องทางต่อไปนี้

- เว็บไซต์สาธารณะและหน้าแบบฟอร์มติดต่อ
- การสมัครสมาชิก การเข้าสู่ระบบ และการเชื่อมต่อบัญชีด้วย LINE
- การขอวัดพื้นที่ การจองบริการ การขอใบเสนอราคา และการสั่งงานบริการดูแลสวน
- การจองเวิร์กช็อป การชำระเงิน และการออกเอกสารประกอบ
- พอร์ทัลลูกค้า พอร์ทัลพนักงาน และพอร์ทัลผู้ดูแลระบบ
- การอัปโหลดรูปภาพ แปลนบ้าน รายงานหน้างาน และเอกสารอื่น
- การแจ้งเตือนผ่านอีเมล LINE และ in-app notification

## 3. ประเภทข้อมูลส่วนบุคคลที่เราเก็บ

### 3.1 ข้อมูลระบุตัวตนและข้อมูลติดต่อ

- ชื่อ นามสกุล
- อีเมล เบอร์โทรศัพท์
- LINE user identifier หรือข้อมูลบัญชี social login ที่เกี่ยวข้อง
- ข้อมูลบัญชีผู้ใช้และบทบาทการใช้งาน เช่น ลูกค้า พนักงาน ผู้ดูแลระบบ

### 3.2 ข้อมูลทรัพย์สินและข้อมูลการให้บริการ

- ที่อยู่บ้านหรือสถานที่ให้บริการ
- ตำแหน่งพิกัดจากแผนที่หรือข้อมูลสถานที่
- รูปถ่ายบ้าน พื้นที่หน้างาน สวน แปลนบ้าน หรือไฟล์ที่ลูกค้าอัปโหลด
- รายละเอียดบริการที่สนใจ ตารางนัดหมาย ประวัติคำขอ ประวัติการวัดพื้นที่ ประวัติการให้บริการ และรายงานผลการปฏิบัติงาน

### 3.3 ข้อมูลธุรกรรมและเอกสาร

- ข้อมูลคำสั่งซื้อ ใบเสนอราคา ใบแจ้งหนี้ ใบเสร็จ และสถานะการชำระเงิน
- ข้อมูลอ้างอิงธุรกรรมจากผู้ให้บริการชำระเงิน
- ข้อมูลการคืนเงิน ยกเลิก หรือเลื่อนนัด

### 3.4 ข้อมูลการใช้งานระบบและความปลอดภัย

- ประวัติการเข้าสู่ระบบ ความพยายามเข้าสู่ระบบที่ไม่สำเร็จ
- IP address, user agent, session identifier, request metadata
- บันทึกเหตุการณ์ในระบบ เช่น การสร้าง แก้ไข อนุมัติ ยกเลิก หรือส่งการแจ้งเตือน
- ข้อมูล error monitoring และ operational telemetry ที่จำเป็นต่อความมั่นคงปลอดภัยและการตรวจสอบย้อนหลัง

### 3.5 ข้อมูลความยินยอมและความพึงประสงค์ในการสื่อสาร

- การยอมรับเงื่อนไขการใช้บริการและนโยบายความเป็นส่วนตัว
- การยินยอมรับข่าวสาร การตลาด หรือโปรโมชัน
- การยินยอมรับการแจ้งเตือนผ่านช่องทางต่าง ๆ เช่น LINE หรืออีเมล
- เวอร์ชันเอกสารที่ยอมรับ วันที่ เวลา ช่องทาง IP และ user agent ของการให้ความยินยอม

## 4. วัตถุประสงค์และฐานกฎหมายในการประมวลผล

### 4.1 เพื่อการเข้าทำสัญญาและการให้บริการ

เราใช้ข้อมูลเพื่อสมัครสมาชิก ยืนยันตัวตน จัดการบัญชี วางแผนงาน วัดพื้นที่ ประเมินราคา จัดตารางทีมงาน ออกเอกสาร ติดตามผล และให้บริการตามที่ลูกค้าร้องขอ โดยอาศัยฐานกฎหมายเรื่องความจำเป็นเพื่อการเข้าทำสัญญาหรือปฏิบัติตามสัญญา

### 4.2 เพื่อปฏิบัติตามกฎหมาย

เราอาจเก็บและใช้ข้อมูลเพื่อออกเอกสารทางบัญชี ภาษี การเก็บหลักฐานทางธุรกรรม การตอบข้อร้องเรียน หรือการปฏิบัติตามคำสั่งจากหน่วยงานที่มีอำนาจตามกฎหมาย

### 4.3 เพื่อประโยชน์โดยชอบด้วยกฎหมาย

เราอาจใช้ข้อมูลเพื่อ

- รักษาความมั่นคงปลอดภัยของระบบ
- ป้องกันทุจริตหรือการใช้งานโดยมิชอบ
- ตรวจสอบคุณภาพการให้บริการและแก้ไขปัญหาเชิงปฏิบัติการ
- เก็บ audit logs และ security logs เพื่อการตรวจสอบย้อนหลัง
- บริหารความต่อเนื่องของบริการและพัฒนาเสถียรภาพของระบบ

### 4.4 เพื่อความยินยอม

ในกรณีการตลาด ข่าวสาร โปรโมชัน หรือการใช้ข้อมูลบางประเภทเกินกว่าที่จำเป็นต่อการให้บริการ เราจะขอความยินยอมจากท่านก่อน และท่านสามารถถอนความยินยอมได้ภายหลังตามช่องทางที่กำหนด

## 5. การเปิดเผยข้อมูลส่วนบุคคล

เราอาจเปิดเผยข้อมูลเท่าที่จำเป็นให้กับผู้ประมวลผลข้อมูลส่วนบุคคลหรือผู้ให้บริการภายนอกที่เกี่ยวข้องกับการดำเนินงานของระบบ เช่น

- ผู้ให้บริการโครงสร้างพื้นฐานและฐานข้อมูล เช่น Supabase
- ผู้ให้บริการชำระเงิน เช่น Stripe, Omise หรือผู้ให้บริการที่ใช้งานจริงในแต่ละบริการ
- ผู้ให้บริการอีเมลและระบบแจ้งเตือน
- LINE และบริการเชื่อมต่อข้อความที่ลูกค้าเลือกใช้
- ผู้ให้บริการแผนที่หรือ geolocation เช่น Google Maps
- ผู้ให้บริการ monitoring และ error reporting เช่น Sentry

เราจะเปิดเผยข้อมูลเท่าที่จำเป็นตามวัตถุประสงค์ และอยู่ภายใต้ข้อตกลงการประมวลผลข้อมูลหรือมาตรการที่เหมาะสมตามความเสี่ยง

## 6. การเก็บ log และการตรวจสอบย้อนหลัง

ระบบนี้มีการเก็บ application logs, audit logs, business event logs และ security logs เพื่อรองรับการให้บริการจริง การสอบทานเหตุการณ์ การติดตามปัญหา และการพิสูจน์ความถูกต้องของกระบวนการสำคัญ เช่น การเปลี่ยนแปลงคำสั่งซื้อ การยืนยันชำระเงิน การส่งแจ้งเตือน และการเข้าถึงข้อมูลสำคัญ

เราออกแบบให้การเก็บ log สอดคล้องกับหลัก data minimization โดยหลีกเลี่ยงการบันทึกข้อมูลลับ เช่น รหัสผ่าน token ลับ หรือข้อมูลบัตรแบบเต็มรูปแบบ และใช้การปกปิดข้อมูลบางส่วนหรือ redaction เมื่อเหมาะสม

## 7. ระยะเวลาการเก็บรักษา

เราจะเก็บข้อมูลตามระยะเวลาที่จำเป็นต่อวัตถุประสงค์ของการให้บริการ ภาระหน้าที่ตามกฎหมาย และการป้องกันข้อพิพาท โดยยกตัวอย่างดังนี้

- ข้อมูลบัญชีลูกค้าและประวัติการให้บริการ: ตลอดอายุความสัมพันธ์ทางบริการและระยะเวลาที่จำเป็นต่อการสนับสนุนหรือข้อพิพาท
- เอกสารทางบัญชีและข้อมูลธุรกรรม: ตามระยะเวลาที่กฎหมายกำหนด
- ข้อมูลความยินยอม: ตลอดระยะเวลาที่อาศัยฐานความยินยอมนั้น และระยะเวลาที่จำเป็นเพื่อพิสูจน์การให้ความยินยอม
- security logs และ audit logs: ตามความสำคัญของเหตุการณ์และนโยบาย retention ภายใน
- ข้อมูลการตลาด: จนกว่าจะถอนความยินยอม หรือหมดความจำเป็นตามวัตถุประสงค์

เมื่อพ้นระยะเวลาที่จำเป็นแล้ว เราจะลบ ทำลาย หรือทำให้ข้อมูลไม่สามารถระบุตัวบุคคลได้ตามความเหมาะสม

## 8. สิทธิของเจ้าของข้อมูล

ภายใต้เงื่อนไขตามกฎหมาย ท่านอาจมีสิทธิ

- ขอเข้าถึงและขอรับสำเนาข้อมูลส่วนบุคคล
- ขอแก้ไขข้อมูลให้ถูกต้อง เป็นปัจจุบัน และสมบูรณ์
- ขอให้ลบหรือทำลายข้อมูลเมื่อไม่มีเหตุจำเป็นทางกฎหมายหรือสัญญาให้ต้องเก็บต่อ
- ขอระงับการใช้ข้อมูลบางกรณี
- คัดค้านการประมวลผลที่อาศัยฐานประโยชน์โดยชอบด้วยกฎหมายในบางกรณี
- ขอถอนความยินยอมที่เคยให้ไว้
- ขอให้โอนย้ายข้อมูลในกรณีที่กฎหมายรองรับ

การใช้สิทธิบางประเภทอาจถูกจำกัด หากขัดต่อภาระหน้าที่ตามกฎหมาย ความมั่นคงปลอดภัย หรือความจำเป็นเพื่อปฏิบัติตามสัญญาที่ยังคงมีอยู่

## 9. การรักษาความมั่นคงปลอดภัยของข้อมูล

เราใช้มาตรการเชิงเทคนิคและเชิงองค์กรตามระดับความเสี่ยงของระบบ เช่น

- role-based access control
- การแยกสิทธิ์การเข้าถึงของลูกค้า พนักงาน และผู้ดูแลระบบ
- audit trail สำหรับเหตุการณ์สำคัญ
- การตรวจสอบความผิดปกติและ error monitoring
- การจำกัดการเข้าถึงข้อมูลสำคัญเฉพาะผู้มีหน้าที่เกี่ยวข้อง
- การใช้ผู้ให้บริการที่สนับสนุนการรักษาความปลอดภัยในระดับที่เหมาะสม

## 10. การโอนข้อมูลไปต่างประเทศ

ในบางกรณีข้อมูลอาจถูกประมวลผลหรือเก็บในระบบของผู้ให้บริการที่ตั้งอยู่ต่างประเทศ เราจะดำเนินการโดยคำนึงถึงมาตรฐานคุ้มครองข้อมูลที่เหมาะสม สัญญาประมวลผลข้อมูล และมาตรการป้องกันที่สอดคล้องกับกฎหมาย

## 11. คุกกี้และเทคโนโลยีที่เกี่ยวข้อง

ระบบอาจใช้คุกกี้หรือเทคโนโลยีคล้ายกันเพื่อคงสถานะการเข้าสู่ระบบ ความปลอดภัย การวิเคราะห์การใช้งาน และการทำงานของฟีเจอร์สำคัญ ทั้งนี้การใช้งานคุกกี้ที่ไม่จำเป็นต่อการให้บริการควรถูกจัดการผ่าน consent layer แยกต่างหากเมื่อมีการใช้งานจริง

## 12. การติดต่อเกี่ยวกับข้อมูลส่วนบุคคล

หากท่านต้องการใช้สิทธิ สอบถาม หรือร้องเรียนเกี่ยวกับข้อมูลส่วนบุคคล กรุณาติดต่อทีมงานผ่านช่องทางบริการของ Xylem Landscape ที่ประกาศใช้จริงในระบบหรือเอกสารติดต่อของบริษัท

## 13. การปรับปรุงนโยบาย

เราอาจปรับปรุงนโยบายฉบับนี้เพื่อให้สอดคล้องกับการเปลี่ยนแปลงของกฎหมาย รูปแบบการให้บริการ หรือโครงสร้างระบบ หากมีการเปลี่ยนแปลงสาระสำคัญ เราจะเผยแพร่เวอร์ชันใหม่และอาจขอให้ผู้ใช้ยอมรับใหม่ตามกรณี
