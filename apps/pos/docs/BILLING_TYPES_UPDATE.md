# 🎯 อัปเดต: รองรับ Billing Types สำหรับบริการ

## ✅ การอัปเดตเสร็จสิ้น

### 🎯 จุดประสงค์
เพิ่มความยืดหยุ่นในการกำหนดรูปแบบการชำระเงินสำหรับบริการแต่ละประเภท:
- 🎨 **ออกแบบ** → รายครั้ง (one-time)
- 🏗️ **ก่อสร้าง** → รายครั้ง (one-time)  
- 🔧 **บำรุงรักษา** → รายเดือน/รายปี (recurring)
- ✨ **ทำความสะอาด** → รายเดือน/รายปี (recurring)
- 🌿 **จัดสวน** → ได้ทั้งสองแบบ (both)

## 📊 Billing Types ที่รองรับ

### 1. **One-time** (รายครั้ง)
- ดำเนินงานครั้งเดียว
- ชำระเงินครั้งเดียวเมื่อเสร็จงาน
- เหมาะสำหรับ: ออกแบบ, ก่อสร้าง, บำบัดต้นไม้

### 2. **Recurring** (รายเดือน/รายปี)
- บริการต่อเนื่อง
- ชำระรายเดือนหรือรายปี (ส่วนลด 15%)
- เหมาะสำหรับ: บำรุงรักษา, ทำความสะอาด

### 3. **Both** (ได้ทั้งสองแบบ)
- ลูกค้าเลือกได้ตามต้องการ
- ยืดหยุ่นสูงสุด
- เหมาะสำหรับ: จัดสวน (ทำครั้งเดียวหรือบำรุงประจำ)

## 🔄 การอัปเดตไฟล์

### Customer Services Page (`app/dashboard/customer/services/page.tsx`)

#### เพิ่ม Interface
```tsx
interface Service {
  // ...existing fields...
  billing_type?: 'one-time' | 'recurring' | 'both';
}
```

#### เพิ่ม Helper Functions
- `getBillingOptions()` - ดูว่าบริการรองรับ billing type ไหน
- `isOneTimeService()` - เช็คว่าเป็นบริการรายครั้งหรือไม่
- `isRecurringService()` - เช็คว่ารองรับ recurring หรือไม่
- `getServiceDetails()` - รายละเอียดบริการตามประเภท

#### UI Changes
1. **Period Toggle**: แสดงตาม billing type ของบริการ
   - One-time: ไม่แสดง toggle, แสดง notice แทน
   - Recurring: แสดงเฉพาะ รายเดือน/รายปี
   - Both: แสดงทั้ง รายครั้ง, รายเดือน, รายปี

2. **Price Calculation**: คำนวณราคาตาม billing type
   - One-time: ราคาเดียว
   - Recurring: รายเดือนหรือรายปี (ส่วนลด 15%)

3. **Service Details**: รายละเอียดแตกต่างตามประเภท
   - Design: โฟกัสการออกแบบ (7 ขั้นตอน)
   - Construction: โฟกัสการก่อสร้าง (7 ขั้นตอน)
   - One-time: ขั้นตอนทั่วไปรายครั้ง (7 ขั้นตอน)
   - Recurring: บริการประจำ (7 ขั้นตอน + real-time tracking)

### Admin Service Creation (`app/dashboard/admin/services/create/page.tsx`)

#### เพิ่ม Billing Type Field
```tsx
interface ServiceFormData {
  // ...existing fields...
  billing_type: 'one-time' | 'recurring' | 'both';
}
```

#### UI Enhancements
1. **Step 2**: เปลี่ยนจาก "หมวดหมู่" เป็น "หมวดหมู่ & รูปแบบ"
2. **Category Cards**: แสดง default billing type ในแต่ละ category
3. **Billing Type Selection**: 
   - Auto-select ตาม category
   - อธิบายแต่ละรูปแบบชัดเจน
   - แสดง impact ที่มีต่อลูกค้า

#### Auto-select Logic
- จัดสวน → Both (default)
- บำรุงรักษา → Recurring  
- ออกแบบ → One-time
- ก่อสร้าง → One-time
- ทำความสะอาด → Recurring
- บำบัดต้นไม้ → One-time

## 🎨 UX/UI Improvements

### 1. **Smart Defaults**
- Category เลือกแล้ว → Auto-set billing type
- แสดงคำอธิบายที่เหมาะสมในแต่ละขั้นตอน

### 2. **Clear Communication**
- แสดงข้อความชัดเจนว่าบริการนี้เป็นแบบไหน
- อธิบาย impact ของการเลือก billing type

### 3. **Consistent Experience**
- UX เดียวกันทั้ง customer และ admin side
- Minimal design ยังคงไว้

## 📈 Business Impact

### 1. **ความยืดหยุ่น**
- รองรับบริการหลากหลายรูปแบบ
- Admin สร้างบริการได้ตรงตามธุรกิจ

### 2. **ความชัดเจน**
- ลูกค้าเข้าใจการชำระเงินได้ง่าย
- ลดความสับสนในการเลือกบริการ

### 3. **การเติบโต**
- รองรับบริการใหม่ ๆ ได้ง่าย
- ขยายธุรกิจได้หลากหลายรูปแบบ
